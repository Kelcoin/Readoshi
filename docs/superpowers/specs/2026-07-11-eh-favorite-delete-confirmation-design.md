# EH 收藏夹同步删除确认设计

## 目标

当“同步删除 E 站收藏夹”全局设置已启用时，在每个会实际删除 LANraragi 归档的确认弹窗中显示“同时从 EH/EX 收藏夹移除”复选框。复选框每次打开默认勾选，用户可以仅针对本次删除取消同步。

## 范围

统一覆盖以下三个现有同步删除入口：

- 首页单个归档删除。
- 首页批量归档删除。
- 重复归档页批量删除。

历史记录、待看记录和其他不删除 LANraragi 归档的确认弹窗不显示该复选框。全局同步开关关闭时，三个入口均不显示复选框，也不执行 EH 收藏夹同步。

## 交互设计

复用现有 `ConfirmDialog`，为其增加可选的内容区域。调用方在全局同步开关启用时传入一个原生复选框，文案为“同时从 EH/EX 收藏夹移除”。

每次打开删除确认弹窗时，本次同步选择重置为 `true`。取消弹窗不会改变全局设置；取消勾选并确认只影响当前单次或当前批次删除。

重复归档页用同一个 `ConfirmDialog` 替换原生 `window.confirm`，从而与首页保持一致的交互和无障碍语义。

## 数据流

1. 用户触发删除，调用方打开确认弹窗并把本次同步选择设为 `true`。
2. 用户可以取消勾选，然后确认删除。
3. 删除处理函数把本次选择传给现有同步步骤。
4. 只有“全局开关启用”且“本次选择勾选”同时成立时，才解析归档中的 EH/EX 地址并调用 Worker `/favorite` 移除收藏。
5. 同步步骤成功或被跳过后，调用 LANraragi `DELETE /api/archives/{id}` 删除归档。

## 错误处理

沿用现有一致性策略：如果用户选择同步且 EH 收藏夹移除失败，则该归档不继续从 LANraragi 删除，并在现有失败提示中报告错误。未勾选同步时，不调用 EH Worker，因此 EH 配置或网络错误不会阻止 LANraragi 删除。

批量删除继续逐项处理；单个归档失败不阻止后续归档尝试，最终沿用现有汇总提示。

## 实现边界

- 不新增依赖。
- 不新增全局设置或持久化本次复选框状态。
- 不重构现有 EH Worker 协议和 LANraragi API 封装。
- 不把尚未支持 EH 同步的推荐组件删除入口纳入本次范围。

## 测试与验证

- 使用 Node 内置测试覆盖“全局开关与本次选择必须同时为真”的决策逻辑。
- 验证三个入口打开时默认勾选，取消勾选后不会调用 EH 同步步骤。
- 验证全局开关关闭时不显示复选框。
- 运行生产构建，确认 React 组件和打包均成功。
