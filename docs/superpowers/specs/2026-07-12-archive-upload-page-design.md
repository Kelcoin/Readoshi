# 归档上传页设计

## 目标

为项目增加独立上传页，让用户从设置面板的“工具”区域进入，并通过以下两种方式向当前 LANraragi 服务器添加归档：

1. 从本地一次选择或拖放多个归档文件上传。
2. 输入一个或多个 URL，通过 LANraragi 下载插件获取归档。

页面沿用现有玻璃面板、按钮、选择框、深浅色主题和移动端响应式风格，不引入新依赖。

## 页面入口与路由

- 设置面板“工具”区域在“打开重复归档检测”旁增加“上传归档”。
- 新路由为 `/?view=upload`，由现有轻量路由系统解析和派发。
- 上传页提供“返回”按钮：优先返回上一界面，没有可返回历史时回到主页。
- 页面任务执行期间离开或刷新时触发浏览器原生确认，避免误中断。

## 本地文件上传

- 页面顶部提供文件选择按钮和拖放区域，两者写入同一任务列表。
- 文件选择支持多选；同一批次按文件名、大小和最后修改时间去重。
- 文件类型提示覆盖 ZIP、CBZ、RAR、CBR、7Z 和 PDF，但最终有效性由 LANraragi 校验。
- 每个文件单独构造 `FormData` 并调用 `PUT /api/archives/upload`。
- 请求沿用现有服务器地址和 Bearer API Key；不得手动设置 multipart 的 `Content-Type`，由浏览器生成 boundary。
- 文件按队列顺序逐项上传，一项失败不会终止其余任务。

## URL 下载

- 多行文本框按“一行一个 URL”解析，忽略空行并按规范化后的完整 URL 去重。
- 只接受 `http:` 和 `https:` URL；无效项在执行前显示为错误。
- 页面从 LANraragi 下载插件接口加载插件并显示项目统一的自定义选择框。
- 默认提供“自动匹配”选项：读取下载插件自带的 URL 匹配正则，逐个编译并对完整 URL 执行匹配，按接口返回顺序选择第一个匹配插件。
- 无效正则会被安全忽略并作为插件加载警告展示；没有正则命中的 URL 要求用户手动指定插件，不轮询执行全部插件。
- 手动选择插件后，该插件处理本批所有有效 URL。
- 每个 URL 单独调用插件执行接口并独立记录结果，一项失败不影响后续任务。

## 状态与反馈

- 本地文件和 URL 共用任务状态模型：`等待`、`处理中`、`成功`、`失败`。
- 页面显示当前任务、已完成数量、总数，以及每项的文件名或 URL 和服务器返回信息。
- 执行期间禁用重复提交，但允许继续查看任务状态。
- 全部任务结束后调用现有搜索缓存清理接口；缓存清理失败不改变已完成任务的成功状态。
- API 错误优先展示服务器返回信息，并延续现有 401 未授权提示格式。

## 代码边界

- `src/pages/UploadPage.jsx`：页面交互、队列执行和结果展示。
- `src/lib/api.js`：下载插件列表、插件执行和 multipart 文件上传请求。
- `src/lib/upload.js`：URL 解析、插件选项规范化、正则编译匹配和任务辅助函数；保持为无浏览器依赖的纯函数以便单元测试。
- `src/lib/navigation.js` 与 `src/App.jsx`：上传页路由。
- `src/pages/Home.jsx`：设置工具入口。
- `src/components/AppGlyphs.jsx`：复用现有图标体系增加上传图标，不引入位图资源。

Worker、配置导入导出、阅读器设置和归档元数据结构不在本次修改范围内。

## 测试与验收

- 单元测试覆盖 URL 换行解析、去重、协议校验、插件列表规范化、首个正则命中、无匹配和无效正则隔离。
- API 测试或可注入 fetch 检查本地上传使用 `PUT`、`FormData` 和现有授权头，且没有错误设置 multipart `Content-Type`。
- 路由测试覆盖 `view=upload` 的解析与导航。
- 运行现有相关单元测试、主题自检和生产构建。
- 验收标准：用户可从设置进入上传页，批量本地上传或批量 URL 下载，每项状态独立可见，失败不会中断整批，完成后可返回原界面。
