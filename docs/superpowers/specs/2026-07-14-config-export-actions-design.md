# 配置导出按钮布局设计

## 目标

导出配置弹窗中，将“复制配置”改为“复制”，并与“关闭”按钮等宽并排。

## 方案

- 复用 `ConfirmDialog` 现有操作栏。
- 为操作栏增加可选的前置操作节点，不改变既有取消、确认按钮行为。
- 导出模式将“复制”作为前置操作，将“关闭”保留为确认操作。
- 两个按钮继续使用 `.confirm-dialog-actions > .btn` 的 `flex: 1`，保持同宽和现有间距。
- 复制成功后按钮文案仍变为“已复制”；失败提示仍在弹窗内显示。

## 范围

只修改 `ConfirmDialog` 与 `ConfigTransferDialog`。不新增组件，不改变导入弹窗、遮罩关闭、Escape 关闭和剪贴板错误处理。

## 验证

- 生产构建通过。
- 导出模式显示“复制”“关闭”两个等宽并排按钮。
- 导入模式按钮布局与行为不变。
