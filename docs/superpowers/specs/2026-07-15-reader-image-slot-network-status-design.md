# 阅读器图片槽位与网络提示设计

## 目标

- 沉浸滑动预览绝不显示错误页或旧页。
- “正在加载”只表示真实网络请求持续超过 180ms；缓存读取和图片解码不显示。
- 普通分页模式在当前页就绪后，按距离优先预加载并预解码相邻页。

## 根因

沉浸模式复用左、当前、右三个原生 `img`。当前页改变时，相邻槽位仍保留上一轮图片，且槽位没有页码身份；新图片解码完成前，下一次半滑会暴露旧图。普通模式后台预加载只缓存 Blob，没有调用浏览器解码。现有加载状态覆盖缓存查询、网络、设置 `src` 和 `decode()` 全阶段，无法区分真实网络等待。

## 设计

### 沉浸图片槽位

每个常驻 `img` 用 `dataset.pageIndex` 记录已完成解码的页号。开始为槽位加载新页时，如果记录页号不等于目标页，立即隐藏并清除页号；解码成功且加载序号仍有效后，再写入目标页号并显示。

滑动晋升相邻预览时，除 `complete`、`naturalWidth`、可见性外，必须校验 `dataset.pageIndex` 等于本次目标页。晋升后同步把当前槽位标为目标页，避免后续 loader 误判。

### 网络加载提示

`resolvePageImageSource()` 接受可选 `onNetworkStart`。该回调只在 `getImage()` 的 fetcher 真正执行时触发，因此内存缓存、Cache API 命中不会触发。

普通模式 `PageImage` 仅在收到 `onNetworkStart` 后启动 180ms 定时器；网络 Blob 返回时立即撤销提示，即使随后还在解码。错误提示保持现状。

沉浸模式只让当前页网络请求控制全屏提示；邻页预加载不控制提示。网络请求超过 180ms 才显示，完成后立即隐藏。

### 普通模式相邻页预解码

保留现有全局队列、当前页 ready 门控和按距离优先索引。`primePageImage()` 在获得共享 object URL 后创建临时 `Image`，设置 `src` 并调用 `decode()`；不支持 `decode()` 时等待 `load`。失败只影响预热，不改变当前页错误状态。

## 约束

- 不新增依赖，不重写全局队列。
- 当前页未 ready 时不启动邻页工作。
- 不降低相邻页预加载数量，不影响 LTR/RTL、双页、Webtoon、冷恢复。
- 保持现有视觉尺寸和动画；只减少错误帧与加载提示出现频率。

## 验证

- 源码回归检查覆盖槽位页号失效/校验、网络回调边界、180ms 网络提示、`primePageImage()` 解码。
