# 首页性能修复设计

## 目标

降低首页封面等待、滚动卡顿和补位动画卡顿，同时保持现有滚动/分页模式、宽卡补位、末行居中、多选、历史恢复及可见卡片动画。

## 根因

- `ArchiveCard` 挂载即请求封面，离屏卡片与可见卡片争抢队列。
- 普通封面只有两个并发槽；长列表导致可见封面排队。
- 无裁剪模式先用额外 `Image` 解码尺寸，再由实际 `<img>` 解码一次。
- 每张宽图识别都会立即增加 Grid 布局版本；Grid 随后遍历全部卡片读取位置。
- `ArchiveGrid` 以 React children 对象为动画依赖；无布局变化的父组件渲染也会触发全表测量。
- 所有离屏卡片完整参与内容布局/绘制。
- 自动加载哨兵和“加载更多”按钮同时存在；按钮进入视口时可能与哨兵竞争。

## 方案

### 封面调度

`ArchiveCard` 内使用一个模块级共享 `IntersectionObserver`。卡片进入视口附近才允许调用现有 `getImage`；预加载边界覆盖纵向约一屏、横向轮播邻近区域。卡片首次进入后保持已加载封面，避免回滚时闪烁。浏览器不支持 `IntersectionObserver` 时维持立即加载。

实际 `<img>` 增加 `loading="lazy"` 和 `decoding="async"`。删除额外 `Image`/`decode()` 路径，宽高比只从实际图片 `onLoad` 读取。图片加载队列总并发从 3 调整为 5，继续保留一个关键任务槽，普通封面最多四个并发。

### Grid 重排

宽度报告仍写入现有 `widthsRef`，但同一动画帧内只增加一次布局版本。组件卸载时取消待执行帧。

FLIP 测量依赖改为稳定布局签名：容器宽度、间距、宽度版本和子项 key 顺序。父组件仅更新无关状态时不再扫描全部卡片。补位造成真实顺序或宽度变化时仍重新测量。

只为当前视口及上下各一屏范围内的移动卡片创建动画；离屏卡片直接落到最终位置。`prefers-reduced-motion` 行为不变。

### 离屏渲染

全部档案 Grid 的直接卡片启用 `content-visibility: auto` 和固定后备块尺寸。浏览器可以跳过远离视口的卡片内容绘制；DOM、卡片顺序、选择状态和滚动高度仍保留，因此不引入列表虚拟化的功能风险。

### 加载入口

支持 `IntersectionObserver` 时，滚动模式只使用哨兵自动加载，并显示已有加载状态；不再同时显示可点击“加载更多”。不支持该 API 时禁用自动观察并保留按钮，确保兼容性。

## 数据流

1. 首页返回档案元数据并渲染卡片占位。
2. 共享观察器标记邻近卡片可加载。
3. 可加载卡片进入现有缓存/请求队列，最多四个普通请求并行。
4. 实际图片加载完成后记录宽高比；同帧宽度变化合并成一次 Grid 重排。
5. Grid 重新补位；只对邻近视口且位置变化的既有卡片执行 220ms 动画。

## 错误与兼容

- 封面缓存、网络失败和重试逻辑保持现状。
- 无 `IntersectionObserver` 时立即加载封面并使用手动按钮。
- 无 Web Animations API 或减少动态效果时直接更新位置。
- 不改变 LANraragi API、筛选、分页、删除、同步和阅读进度链路。

## 非目标

- 不引入列表虚拟化或新依赖。
- 不改变卡片视觉尺寸、补位算法、动画时长或浏览模式默认值。
- 不改真实服务器配置或数据。
